Share Your Drive Catalogues with Anyone on Your Team

Published July 18, 2026 • 6 min read

Let's save you some time: DriveVault is the easiest way to share an external hard drive catalogue with someone on your team, wherever they are in the world. Scan a drive once, and you can send its entire contents - searchable, browsable, completely offline - to anyone in seconds.

There are three ways to do it in DriveVault, each suited to a slightly different situation. Here they are.

📦
Method 1

Export and Import Full Catalogues

For teams where everyone uses DriveVault

This is the deepest form of sharing. Export a complete drive catalogue as a .drivevault file and send it to anyone on your team. When they import it into their own DriveVault, they can browse and search the drive exactly as if they had scanned it themselves - every folder, every file, every piece of metadata, every tag and rating you have applied. The drive does not need to be anywhere near them.

A .drivevault file is self-contained and compressed - transfer it by email, AirDrop, shared cloud folder, or USB stick. The recipient gets full search, full browsing, and full offline preview capabilities instantly.

What travels with the catalogue:

  • Complete file and folder structure
  • All file sizes and dates
  • EXIF, video, and audio metadata
  • Star ratings you have applied
  • Custom tags and notes
  • Offline preview thumbnails
  • Drive statistics
  • Instant search index

📊
Method 2

HTML Reports

For clients, stakeholders, or anyone without DriveVault

Not everyone on a project uses DriveVault - and some people just need to see what is on a drive, not browse it interactively. For those situations, DriveVault generates a beautiful, self-contained HTML report that opens in any web browser on any device - Mac, Windows, iPhone, anything.

These are not plain text file lists. They are visual dashboards with interactive charts showing file type breakdowns, storage distribution, category summaries, and a searchable, sortable file table. They open offline, require no software, and look professional enough to send directly to a client or present in a meeting.

A timestamped HTML report is also a permanent record - a snapshot of a drive at a specific point in time, useful for audit trails, delivery documentation, and archiving.

Client deliverables Stakeholder reviews Archive documentation Audit trails Project sign-off
📋
Method 3

CSV Export

For data workflows, inventory systems, and analysis

Sometimes a team needs raw data rather than a visual interface. A production coordinator building an asset inventory in Excel. A post-production supervisor importing drive contents into a project management database. A producer running analysis across multiple drive libraries to find duplicate files or calculate storage usage.

DriveVault exports a clean, standard CSV file with every file on a drive listed as a row - filename, full path, size in bytes, human-readable size, last modified date, and file category. It opens in Excel, Google Sheets, Numbers, or feeds directly into any database or automation workflow.

This is also the most portable format of the three - a CSV file is readable by virtually any software and will remain so indefinitely, making it ideal for compliance documentation or long-term archive records.

No cloud, no accounts, no tracking. All three sharing methods work entirely through direct file transfer - email, AirDrop, shared folder, USB stick, whatever you normally use. DriveVault never sees your data, never sends anything to a server, and never requires anyone to create an account. The catalogue is yours, and you control who it goes to.
